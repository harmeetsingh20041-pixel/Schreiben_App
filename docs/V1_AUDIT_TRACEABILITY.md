# V1 Audit Traceability Ledger

This ledger is the launch source of truth for the July 2026 repository and live
workflow audit. Every audit finding is represented once below. Update the same
row when implementation or evidence changes; do not remove rows.

Statuses:

- **Open** — no verified implementation yet.
- **In progress** — implementation exists locally but its full release gate has not passed.
- **Foundation** — the automated test or delivery mechanism exists; the product finding remains open.
- **Verified** — implementation and the named regression test both passed with recorded evidence.

Evidence must identify a test result, commit, deployment, or dated manual check.
Code presence alone is not enough to mark a finding Verified.

Severity is classified separately in
`quality/v1-audit-severity.json` so release checks cannot trust self-reported
pilot counts. `pnpm audit:verify` requires exact ID parity and valid evidence
fields; `pnpm audit:verify:launch` fails until every classified finding is
Verified. P0 means a launch-blocking security/data-integrity failure, P1 means
a launch-blocking core reliability, quality, accessibility, or operational
failure, and P2 is a lower-impact issue that still must be Verified before the
pilot.
